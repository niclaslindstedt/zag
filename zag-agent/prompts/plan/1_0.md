## Instructions

Create a structured implementation plan in Markdown. The plan should be actionable and specific to this codebase.

### Plan Structure

1. **Summary** — One paragraph describing what will be built and why.

2. **Affected Files** — List every file that will be created or modified, with a one-line description of the change.

3. **Implementation Steps** — Numbered, ordered steps. Each step should:
   - Name the file(s) to change
   - Describe the specific code changes (functions, structs, modules, etc.)
   - Note any dependencies on prior steps

4. **Testing Approach** — How to verify the implementation:
   - Unit tests to add
   - Integration tests if applicable
   - Manual verification steps

5. **Risks & Considerations** — Potential issues, edge cases, breaking changes, or architectural concerns.

### Guidelines

- Be specific: reference actual file paths, function names, and module structure from the codebase.
- Keep steps small and independently verifiable where possible.
- Output clean Markdown — no code fences wrapping the entire plan, no preamble.
